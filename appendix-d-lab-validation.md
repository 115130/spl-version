# 附录 D · 全书实验验收路线

> 每章完成后，不只问“代码编译了吗”，还要问“硬件是否出现了可重复观察的现象”。

| 阶段 | 章节 | 最小验收标准 |
|---|---|---|
| 准备 | 前言、0.5、0 | ST-Link 被识别；ZET6 工程能编译、烧录、闪灯 |
| 基础 | 1–4 | 能解释程序从 Flash 启动；能让 LED、按键按预期工作 |
| 时序 | 5–7 | 串口波特率正确；定时器周期和 PWM 占空比可观察 |
| 通信 | 8 | USB-TTL 能稳定收日志并交互命令 |
| 模拟 | 9 | 电位器、NTC 或电池分压值与万用表读数基本一致 |
| 总线 | 10–12.5 | I2C 有 ACK；SPI/SDIO 能读写；RS485 CRC 和收发方向正确 |
| 存储 | 13 | 掉电后日志策略明确，文件系统错误可恢复 |
| RTOS | 14–16 | 任务栈高水位、队列溢出、互斥错误都有可观察指标 |
| 无线 | 17–20 | 模块可配网；断网后不会卡住；TCP 收包按字节流处理 |
| 云端 | 21–24 | MQTT/HTTP 有明确连接状态、重连、错误码和消息边界 |
| 项目 | 25–27 | 每个项目按里程碑完成，单个模块故障不会拖垮全系统 |
| 工程 | 28–30 | 能用 GDB 定位故障；测过功耗；构建和发布可复现 |

## 完成前检查

- [ ] 接线图与代码中的引脚一致；
- [ ] 记录过至少一种可观察证据：日志、LED、波形、万用表读数；
- [ ] 出错时先检查过供电、GND、引脚与时钟；
- [ ] 没有把真实密码或密钥写进仓库；
- [ ] 知道本实验的限制和下一章要解决的问题。
