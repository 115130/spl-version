# 第 2 章 · STM32F103 硬件概览

> **本章产出**：理解存储器映射（为什么 `*(volatile uint32_t *)0x40010800` 能控制硬件）、看懂时钟树、理解引脚复用
>
> **用到项目的哪里**：每一个项目都要配置时钟、分配引脚、操作外设寄存器。本章是后面所有内容的「地图」

> **与 HAL 版的区别**：硬件概念完全相同，但本章用 SPL 工程和 RM0008 参考手册代替 CubeMX 来讲解时钟和引脚。

## 2.1 ARM Cortex-M3 内核

STM32F103 的大脑是一颗 **ARM Cortex-M3** 处理器内核。

但这里面有一个容易误解的地方，先说清楚：

- **ARM 公司**不生产任何芯片。ARM 只**设计处理器内核的「图纸」**（Verilog/VHDL 源代码，叫 IP 核），然后把这些图纸授权给其他芯片公司。ARM 赚钱靠的是**卖 IP 授权 + 收版税**，不靠卖芯片。
- **ST 公司**（意法半导体）从 ARM 买了 Cortex-M3 的 IP 授权，拿到了 CPU、NVIC、SysTick、调试单元这些「核」的设计图纸。然后 ST 自己**设计了外围电路**——GPIO、USART、SPI、I2C、ADC、Flash 控制器、SRAM、时钟树、总线矩阵——把这些和 Cortex-M3 核整合成一张完整的芯片设计图。
- **台积电**（或其他晶圆厂）拿着 ST 的设计图在硅片上刻出真正的芯片。

所以「壳」和「核」都只是**设计图纸**——ARM 画了核的部分，ST 画了壳的部分，台积电把整张图纸变成硅片。

这样做的好处是：只要你学会了 STM32F103 的用法，你就同时学会了 ARM Cortex-M3 内核的用法。NXP、TI、Atmel 也有用 Cortex-M3 的芯片——内核一样，外设不同，编程模型通用。

### 关键特性

| 特性 | 说明 | 对比你学过的计组 |
|------|------|-----------------|
| **RISC** | 精简指令集，大部分指令单周期执行 | ARM 就是商业上最成功的 RISC |
| **哈佛架构** | 指令和数据走不同的总线 | 计组课本的五级流水线通常用哈佛架构 |
| **三级流水线** | 取指→译码→执行，可同时进行 | Cortex-M3 简化了 MIPS 的五级 |
| **Thumb-2 指令集** | 混合 16 位和 32 位指令，兼顾密度和性能 | 不需要学汇编，但知道有这个东西 |
| **NVIC** | 嵌套向量中断控制器，支持最多 240 个中断 | 工业级中断实现，比书本上的最简模型复杂得多 |
| **MPU/MMU** | F103 没有 MMU，没有虚拟内存 | 所以跑不了 Linux |

### 寄存器组

Cortex-M3 有 16 个 32 位寄存器：

| 寄存器 | 用途 |
|--------|------|
| R0-R12 | 通用寄存器 |
| R13 (SP) | 栈指针（MSP 主栈 / PSP 进程栈） |
| R14 (LR) | 链接寄存器（存函数返回地址） |
| R15 (PC) | 程序计数器（当前执行的指令地址） |

还有特殊寄存器：xPSR（程序状态寄存器）、PRIMASK/FAULTMASK/BASEPRI（中断屏蔽）、CONTROL（特权级/栈选择）。

**不需要背。** 编译器替你管理。知道有 16 个寄存器就够了——后面 Debug 遇到 Hard Fault，查看寄存器值是重要手段。

## 2.2 存储器映射——整个芯片的「地址地图」

这是本章**最重要**的一节。

假设你有一张 4GB 的内存空间（32 位地址总线，2³² = 4,294,967,296），STM32 的设计师把这张「白纸」画成了一张地图：

```
0xE000_0000 ┌──────────────────┐
            │  Cortex-M3 内部   │  ← NVIC、SysTick、调试组件
0xC000_0000 ├──────────────────┤  (保留不用)
0xA000_0000 ├──────────────────┤  (保留不用)
0x8000_0000 ├──────────────────┤  (保留不用)
0x6000_0000 ├──────────────────┤  ← FSMC（外部存储器）
0x4000_0000 ├──────────────────┤
            │   外设寄存器区     │  ← GPIO、USART、SPI、I2C... 所有外设的寄存器
            │   (APB1/APB2/    │     每一个 SPL 函数都在操作这里的地址
            │    AHB 总线外设)  │
0x2000_0000 ├──────────────────┤
            │   SRAM (64KB)    │  ← 全局变量、堆、栈
0x1FFF_0000 ├──────────────────┤
            │   系统存储器       │  ← ST 出厂预烧的 Bootloader
0x0800_0000 ├──────────────────┤
            │   Flash (512KB)  │  ← 你的程序烧在这里
0x0000_0000 └──────────────────┘
```

**关键洞察**：在嵌入式世界里，**硬件外设的「控制接口」就是特定地址上的寄存器**。操作 GPIO 就是读写 `0x4001_0C00` 附近的几个 32 位地址；操作 USART 就是读写 `0x4001_3800` 附近的地址。

### 一个具体例子：GPIOB 的寄存器地址

翻开 STM32F103 参考手册（RM0008），查 GPIOB：

| 寄存器 | 偏移 | 绝对地址 | 功能 |
|--------|------|---------|------|
| GPIOB_CRL | 0x00 | 0x4001_0C00 | 配置 Pin0-Pin7 的模式 |
| GPIOB_CRH | 0x04 | 0x4001_0C04 | 配置 Pin8-Pin15 的模式 |
| GPIOB_IDR | 0x08 | 0x4001_0C08 | 读取引脚输入电平 |
| GPIOB_ODR | 0x0C | 0x4001_0C0C | 设置引脚输出电平 |
| GPIOB_BSRR | 0x10 | 0x4001_0C10 | 原子位操作（写 1 置位/复位） |

所以你写 `GPIO_ResetBits(GPIOB, GPIO_Pin_5)` 时，最终会变成对 `0x4001_0C10` 的一次 32 位写入——向 BSRR 的 bit 21 写 1，复位 PB5。

**这个寄存器地址表在你的 SPL 工程里就是 `inc/stm32f10x.h` 中的宏定义**：

```c
#define GPIOB_BASE          (APB2PERIPH_BASE + 0x0C00)
#define GPIOB               ((GPIO_TypeDef *) GPIOB_BASE)
```

你顺手打开 `stm32f10x.h` 翻一下这些宏——不用背，留个印象能找就行。

## 2.3 总线矩阵——指令和数据为何能同时走

STM32F103 内部有多条总线，通过一个**总线矩阵**（Bus Matrix）连接起来：

```
   Cortex-M3 内核
   ┌─────────────┐
   │ I-Code 总线  │ ───→ Flash (取指令)
   │ D-Code 总线  │ ───→ Flash (读常量数据)
   │ System 总线  │ ───→ SRAM + 外设
   └─────────────┘
          │
    总线矩阵 (Bus Matrix)
          │
   ┌──────┼──────┐
   ↓      ↓      ↓
  Flash  SRAM  AHB→APB 桥 → 外设
```

**CPU 可以从 Flash 取指令的同时，从 SRAM 读写数据**。两条通路不冲突。这就是哈佛架构相比冯·诺依曼架构（指令和数据共用一条总线）的核心优势。

### AHB 和 APB —— 为什么芯片里有快慢两条路

STM32 内部有两条不同规格的总线：

**AHB（Advanced High-performance Bus）—— 高速公路**

- 跑 72MHz
- 挂高速设备：CPU 核、Flash 接口、SRAM 接口、DMA 控制器
- CPU 通过 AHB 直接读 Flash 和 SRAM

**APB（Advanced Peripheral Bus）—— 省道**

- 又分 APB1（最大 36MHz）和 APB2（最大 72MHz）
- 挂所有外设：GPIO、USART、SPI、I2C、ADC、定时器
- AHB 和 APB 之间有一道「AHB-to-APB Bridge」——CPU 访问外设时，请求从 AHB 经过这道桥转到 APB

**为什么搞两条，不把所有东西直接挂 AHB？**

1. **省电**。APB 外设大部分时候不用，时钟可以单独关掉。你 `RCC_APB2PeriphClockCmd(..., DISABLE)` 的时候，那个 APB 分支的功耗就降了
2. **信号完整性**。72MHz 的高速信号跑全芯片，噪声大、功耗高、物理布线难。把低速外设放到独立的慢总线上，物理设计简单得多

**APB1 和 APB2 的分工**

| | APB2（高速） | APB1（低速） |
|---|---|---|
| 最高频率 | 72MHz | 36MHz |
| 挂的外设 | GPIO、USART1、SPI1、ADC1-2、TIM1、AFIO | TIM2-7、USART2-3、I2C1-2、SPI2-3、DAC、USB |
| 开时钟函数 | `RCC_APB2PeriphClockCmd()` | `RCC_APB1PeriphClockCmd()` |

为什么 USART1 在 APB2 上跑 72MHz，USART2/3 在 APB1 上只跑 36MHz？ST 的设计考量：**USART1 是主力串口**——调试、高速通信都用它，挂高速总线。USART2/3 是辅助串口，36MHz 够了。同理：SPI1 在 APB2、SPI2 在 APB1。

**TIM2-7 有一个特殊规则**：虽然挂在 APB1（36MHz）上，但如果 APB1 预分频不是 /1（默认 /2），定时器时钟自动 ×2，所以 TIM2-7 实际上也能跑 72MHz。ST 的设计师专门给定时器开了后门——计时的精度不能妥协。

> **一句话**：AHB 是高速公路，连 CPU/内存/DMA。APB 是省道，连外设。APB2 是快省道（GPIO/USART1/SPI1），APB1 是慢省道（普通定时器/I2C/辅助串口）。每次你用 `RCC_XXXPeriphClockCmd()` 开外设时钟，就是在给这条省道上的某个「电闸」合上。

## 2.4 时钟树

### 为什么需要时钟

数字电路是同步的——所有的寄存器、触发器都在同一个「心跳」下工作。这个心跳就是**时钟信号**（CLK）。

STM32F103 的时钟树：

```
        HSI (内部 8MHz RC) ──┐
        HSE (外部 8MHz 晶振) ─┤
                             ├── PLL (锁相环) ──×9── 72MHz (SYSCLK)
        LSI (内部 40kHz) ────┤
        LSE (外部 32.768kHz) ─┘

    SYSCLK (72MHz)
        ├── AHB 分频器 ── HCLK (72MHz) ──→ CPU、SRAM、DMA、FSMC
        │       └── APB1 分频器 (/2) ── PCLK1 (36MHz) ──→ TIM2-7、USART2-5、I2C、SPI2/3
        └── APB2 分频器 (/1) ── PCLK2 (72MHz) ──→ GPIO、TIM1、USART1、SPI1、ADC
```

关键点：

1. **上电默认用 HSI**（内部 8MHz），然后 `SystemInit()` 尝试切到 HSE+PLL 跑 72MHz。如果 HSE 失败，退回 HSI
2. **不同外设挂在不同总线上**，操作前必须打开对应的时钟——这就是 `RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE)` 的作用
3. **APB1 最大 36MHz，APB2 最大 72MHz**
4. **SysTick 用 HCLK 或 HCLK/8**，给延时和 RTOS 提供时基

### 在 SPL 工程中看时钟

SPL 没有 CubeMX 的图形化时钟配置。你在 `lib/system_stm32f10x.c` 的 `SystemInit()` 函数末尾能看到：

```c
void SystemInit(void)
{
    // ... 复位各时钟 ...
    SetSysClock();   // ← 根据你的宏定义，选一个时钟配置函数
}
```

`SetSysClock()` 里根据 `SYSCLK_FREQ_*` 宏来选——如果你没有定义任何 `SYSCLK_FREQ_*`，就保持上电默认的 HSI 8MHz。后面第 5 章会教你怎么配到 72MHz。

**查看完整时钟树的权威来源**：参考手册 RM0008 第 7 章「Clock Tree」的图——那是 ST 画的最精确的时钟拓扑。

## 2.5 引脚功能复用

大多数 GPIO 引脚都可以「复用」为外设功能：

```
以 PA9 为例：
   默认：GPIOA Pin9（普通输入输出）
   复用：USART1_TX（串口 1 发送）
   重映射：可以重映射到 PB6（通过 AFIO 寄存器）

以 PA2 为例：
   默认：GPIOA Pin2
   复用：USART2_TX 或 TIM2_CH3 或 ADC12_IN2
```

**同一时间只能选一个功能。** 底层通过配置 **GPIOx_CRL/CRH** 寄存器中的 CNF 位来实现（第 3 章细讲）。SPL 里 `GPIO_Init()` 帮你处理了这些位操作。

### 重要：调试引脚

PA13（SWDIO）、PA14（SWCLK）默认是 SWD 调试接口。如果你把它们配成普通 GPIO，就会**失去调试能力**——ST-Link 连不上。除非你确定不需要调试，否则不要动这两个引脚。

### ZET6 的引脚更丰富

你手上的 STM32F103ZET6 有 144 个引脚，比书中假定的 C8T6（48 引脚）多得多。好处：GPIO 充足，几乎不需要考虑引脚冲突。坏处：需要自己翻阅 ZET6 的数据手册（不是参考手册 RM0008，是 STM32F103xC/xD/xE 的 datasheet）来确认每个引脚的复用功能表。

## 2.6 动手：翻一天数据手册

学会读数据手册是嵌入式工程师的**核心能力**。现在做一个练习：

### 任务：找到「USART1 的寄存器基地址」

1. 打开 STM32F103 参考手册（RM0008，ST 官网免费下载）
2. 目录中找到「Memory Map」章节
3. 顺着找到 USART 相关的外设地址

你会找到：

```
0x4001_3800 - 0x4001_3BFF  USART1
```

所以 USART1 的基地址是 `0x4001_3800`。在这个基地址之上，各个寄存器有自己的偏移量（SR 状态寄存器、DR 数据寄存器、BRR 波特率寄存器...），后面第 8 章会用到。

**这个练习的目的**：不是让你背地址，而是让你知道**信息在哪里**。当你用一个新的外设时，知道去 RM0008 的哪个部分找。

## 2.7 本章要点

- STM32F103 = ARM Cortex-M3 内核 + ST 设计的壳（外设 + 总线 + Flash/SRAM）
- 4GB 地址空间：Flash(0x0800_0000)、SRAM(0x2000_0000)、外设(0x4000_0000)、内核(0xE000_0000)
- 哈佛架构 = 取指和数据通路分开 → 可以同时进行
- 时钟如心跳：上电默认 HSI 8MHz，`SystemInit()` 尝试切到 PLL 72MHz
- SPL 通过 `RCC_APB2PeriphClockCmd()` 开关各外设时钟
- 引脚可复用：PA9 可以是 GPIO，也可以是 USART1_TX，同一时间只能选一个
- 遇到问题翻参考手册 RM0008，不需要背
