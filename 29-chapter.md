# 第 29 章 · 低功耗设计：先测量，再进入睡眠（SPL版）

> **本章产出**：理解 Sleep、Stop、Standby 的区别；能为 STM32F1 设计一次可恢复的 Stop 模式唤醒流程。
>
> **前置知识**：第 5 章时钟、第 6 章中断，以及第 28 章调试方法。
>
> **用在哪**：电池供电的传感器节点、低频采样设备、待机控制器。

---

## 29.1 低功耗不是调用一个函数就结束

真正的功耗由整块系统决定：

~~~text
MCU 时钟 + 外设时钟 + 传感器 + WiFi/BLE 模块 + LED + 电源转换器
~~~

如果 WiFi 模块持续联网、OLED 常亮、LED 常亮，即使 MCU 进入低功耗，整机续航也不会明显改善。第一步永远是测量基线电流，再逐项关闭模块。

## 29.2 三种常见模式

| 模式 | CPU | 外设/时钟 | 唤醒后 | 典型用途 |
|---|---|---|---|---|
| Sleep | CPU 停止 | 多数时钟继续 | 从下一条指令继续 | 短时间等待中断 |
| Stop | 主时钟停止 | 大部分高速时钟关闭 | 需恢复时钟与外设 | 周期采样 |
| Standby | 接近复位状态 | 保留极少部分 | 类似复位启动 | 长时间待机 |

不同芯片和板子细节不同，任何低功耗设计都要回到对应参考手册验证。

## 29.3 先选择唤醒源

进入 Stop 前必须知道“谁来叫醒它”。常见选择：

- RTC Alarm：每 10 分钟采样一次；
- 外部按键或门磁：通过 EXTI 唤醒；
- WKUP 引脚：从 Standby 唤醒；
- 某些外设事件：取决于芯片和时钟配置。

WiFi 的 UART RX 并不自动等于可靠唤醒源。确认模块、引脚、EXTI 和低功耗模式组合后再设计。

## 29.4 一个明确的 Stop 恢复接口

第 5 章应把 72 MHz 时钟配置封装为一个明确函数，例如 Clock_Init72MHz。不要在 SPL 章节中调用没有定义的 SystemClock_Config。

~~~c
void Clock_Init72MHz(void);       /* 来自你的时钟驱动 */
void Periph_ResumeAfterStop(void); /* 重新配置依赖时钟的外设 */

void EnterStopMode(void)
{
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_PWR, ENABLE);

    PrepareWakeupSource();  /* 例如 RTC 或 EXTI */
    FlushPendingLog();

    PWR_EnterSTOPMode(PWR_Regulator_LowPower, PWR_STOPEntry_WFI);

    /* 从中断唤醒后，系统通常回到低速时钟 */
    Clock_Init72MHz();
    Periph_ResumeAfterStop();
}
~~~

不要假设所有外设状态都会自动恢复。UART 波特率、I2C、SPI、SysTick 和外部模块的状态都应在测试板上逐一确认。

## 29.5 传感器节点的工作周期

~~~text
唤醒
  → 恢复时钟
  → 供电并初始化传感器
  → 采样
  → 本地记录 / 条件上报
  → 关闭传感器和不必要外设
  → 设置下一次唤醒
  → Stop
~~~

如果网络上报耗时很长，通常不要每次采样都联网。可以多次本地缓存，再批量连接发送，以换取更长续航。

## 29.6 测试方法

| 测试 | 目的 |
|---|---|
| 运行态电流 | 找到最大的持续耗电源 |
| Stop 电流 | 验证 MCU 和板级外围是否真的降下去 |
| 唤醒成功率 | 连续数百次休眠/唤醒不丢功能 |
| 时钟恢复 | 唤醒后 UART 波特率、I2C、SysTick 正常 |
| 数据完整性 | 休眠前后 seq 和日志不乱 |

万用表适合看平均电流，示波器或电流分析仪更适合看短暂峰值。记录“电压、模式、外设状态、测得电流”，不要只记录一个漂亮数字。

## 29.7 常见误区

- 只调用 WFI，却没有关闭任何外设；
- 进入 Stop 后忘记恢复时钟；
- 用延时等待外设，却让 CPU 忙等；
- 忽略开发板上的稳压器、LED 和 USB-TTL 的静态功耗；
- 没有准备唤醒源，设备“睡死”。

## 29.8 本章练习

1. 让 LED 每 10 秒闪一次，其余时间进入 Sleep；
2. 改为 RTC 或 EXTI 唤醒的 Stop 实验；
3. 对比三种状态下的平均电流；
4. 写一张表，列出每个外设进入休眠前的关闭与恢复动作。

## 29.9 本章要点

- 低功耗是系统工程，先测量再优化；
- Sleep、Stop、Standby 的唤醒成本不同；
- Stop 后必须明确恢复时钟和依赖外设；
- 唤醒源与模式必须作为同一个设计问题；
- 能连续验证数百次的休眠/唤醒，才算可靠。
