# 第 0.5 章 · 硬件零基础生存手册（ZET6 版）

> **本章产出**：在还没写第一行外设代码前，能安全识别自己的开发板、知道哪些线能接、哪些线不能接，并能让 ST-Link 与开发板被电脑正确识别。
>
> **本书固定硬件基线**：STM32F103ZET6 开发板，STM32F10X_HD，512KB Flash / 64KB SRAM。
>
> **写给谁**：你会写前后端、能读 C，但从没接过杜邦线、没量过电压，也不知道 GND 是什么。
>
> **开始前**：本章不要求写代码；目标是安全上电、识别接口并建立“先量电压、再连信号线”的习惯。
>
> **最小验收**：能在断电状态下指出 ST-Link 的 SWDIO/SWCLK/GND、USB-TTL 的 TX/RX/GND，以及开发板的 3.3V 与 GND。

---

## 0.5.1 先建立一个正确的心智模型

软件里的函数调用发生在抽象层中；硬件里的每一次“调用外设”，最终都会落到一根引脚、一条电路和一个电压状态上。

可以用下面的类比快速入门：

| 软件概念 | 硬件中的对应 |
|---|---|
| API | 外设寄存器和驱动函数 |
| 服务进程 | MCU 上持续运行的主循环或任务 |
| 日志 | UART 串口输出 |
| 网络连接 | UART、I2C、SPI、RS485 等物理连线 |
| 配置错误 | 引脚接错、电压不对、时钟没开、外设没有供电 |
| 线上排障 | 看串口、量电压、看逻辑波形、读寄存器 |

硬件最重要的差别是：程序写错通常可以重启，接线或电压错了可能损坏器件。因此先学“安全地失败”。

## 0.5.2 本书只支持哪块板

正文和所有工程配置都以 **STM32F103ZET6** 为唯一基线：

~~~text
芯片：STM32F103ZET6
内核：Cortex-M3
主频：最高 72 MHz
Flash：512 KB
SRAM：64 KB
封装：144 引脚
SPL 宏：STM32F10X_HD
启动文件：startup_stm32f10x_hd.s
~~~

不要把其他教程中的 C8T6、MD 启动文件或 64KB Flash 链接脚本直接复制到本书工程中。它们也许能编译，但会让启动文件、内存大小和引脚假设彼此矛盾。

芯片型号只能告诉你“这个引脚**可以**做什么”，不能告诉你“开发板**已经**把它接到哪里”。第一次使用一块板时，先完成 [板卡资源约定](./board-zet6-profile.md) 第 2 节的六项记录；尤其不要根据“某品牌常见板”猜 LED、USB 串口或板载 Flash 的引脚。

## 0.5.3 必须认识的六个词

| 词 | 先这样理解 |
|---|---|
| 电压 V | 两点之间的电势差；数字电路常用 3.3V 作为高电平参考 |
| 电流 A | 电荷流动的量；过大可能损坏 LED、GPIO 或模块 |
| GND | 全系统共同的参考点；两个模块通信时通常必须共地 |
| GPIO | MCU 可读、可写或复用为外设功能的引脚 |
| 3.3V 逻辑 | ZET6 GPIO 的安全数字电平基线 |
| 限流电阻 | 限制 LED 等器件电流，避免直接短接电源 |

初学阶段请记住两条硬规则：

1. **不要把 5V 数字信号直接接到 ZET6 的普通 GPIO。**
2. **任何两个要通信的模块，先确认它们是否共地。**

补充一条测量纪律：万用表的通断/电阻档只在**断电**时使用；电压档才在上电后使用。黑表笔先固定到已确认的 GND，再用红表笔测目标点。这样既减少短路风险，也使每次读数有共同参考。

## 0.5.4 上电前检查

第一次接线时，只做最少的事情：

~~~text
开发板 ← 电源或 USB
开发板 ← ST-Link（SWDIO、SWCLK、GND，按板卡丝印连接）
电脑   ← ST-Link USB
~~~

注意：

- 以你手上开发板的丝印和原理图为准；不要只凭线的颜色接线。
- 不要同时从多个来源给板子供电，除非板卡说明明确支持。
- ST-Link 的 3.3V 引脚是否需要连接，取决于开发板供电方式；不确定时先查板卡说明。
- 接外部模块前，先确认模块的供电电压与 IO 电平。模块“可接 5V 供电”不代表它的 TX/RX 可以输出 5V。
- 调试器和 USB-TTL 的 GND 是信号参考；它们的 3.3V 输出不是“看到就应该接”的电源线。先只接信号所需的 GND，按板卡供电方案决定是否接电源。

## 0.5.5 UART 的第一条接线规则

后面第 8 章会用 USB-TTL 查看日志。最小接线如下：

~~~text
ZET6 USART_TX  ───→ USB-TTL RX
ZET6 USART_RX  ←─── USB-TTL TX
ZET6 GND       ───── USB-TTL GND
~~~

TX 和 RX 要交叉；GND 必须相连。若串口乱码，先检查：

1. USB-TTL 是否是 3.3V 电平；
2. TX/RX 是否交叉；
3. 两边波特率是否一致；
4. 时钟配置是否正确；
5. 是否已经共地。

## 0.5.6 你需要的最小工具

| 工具 | 用途 | 第一阶段要会什么 |
|---|---|---|
| ST-Link | 烧录与 SWD 调试 | 能被 OpenOCD 识别 |
| USB-TTL | 看 UART 日志 | 能收到 Hello 或错误信息 |
| 万用表 | 看电压、通断、电流 | 先学量 3.3V 与 GND |
| 杜邦线 | 临时连接模块 | 不靠颜色，靠引脚标签 |
| 面包板 | 搭简单电路 | 理解同一排孔是否相通 |

逻辑分析仪和示波器非常有用，但不是第一天的必需品；第 28 章会告诉你什么时候该用它们。

## 0.5.7 首次健康检查

在安装第 0 章的软件前，先完成这张清单：

- [ ] 开发板没有发烫、焦味或异常闪烁；
- [ ] 板卡供电指示灯正常；
- [ ] ST-Link 被 Linux 识别；
- [ ] SWDIO、SWCLK、GND 的连接与板卡丝印一致；
- [ ] 没有把 5V 直接接入普通 GPIO；
- [ ] 外部模块尚未接入，或已确认电压与 GND。
- [ ] 已在板卡资源表写下 LED、按键和 USB 串口的实际引脚；未知项仍标为“待确认”。

发现问题时先断电，再检查接线。不要一边带电一边拔插不确定的电源线。

## 0.5.8 最常见的四类现象

| 现象 | 优先检查 |
|---|---|
| OpenOCD 找不到芯片 | ST-Link、SWDIO/SWCLK、GND、目标板供电 |
| 烧录成功但程序没跑 | 启动文件、链接脚本、时钟、复位脚 |
| 串口没有输出 | TX/RX、共地、波特率、代码是否真的执行到 printf |
| 模块一接就复位 | 供电能力不足、GND 没共地、模块电流峰值过大 |

## 0.5.9 动手：第一次安全接线验收

按下面的顺序做，不要跳步：

1. **断电观察**：在板卡丝印上找到 3.3V、GND、SWDIO、SWCLK、USART1 TX/RX；不确定就查原理图。
2. **只接调试接口**：连接 ST-Link 的 SWDIO、SWCLK、GND；目标板的供电方式以板卡说明为准。
3. **上电后测量**：万用表黑表笔固定在 GND，红表笔量 3.3V；只有读数合理且板子不发烫时，才继续。
4. **再接 USB-TTL**：先接 GND，再交叉接 TX/RX；USB-TTL 必须使用 3.3V 逻辑。
5. **记录结果**：写下你的板卡型号、供电方式和实际连接的引脚，并填入 [板卡资源约定](./board-zet6-profile.md)。后面每章的接线都以这份记录为基准。

| 现象 | 首先停止什么 | 优先检查 |
|---|---|---|
| 板子发烫或闻到异味 | 立即断电 | 电源极性、短路、外设供电 |
| ST-Link 找不到目标 | 不反复插拔电源 | GND、SWDIO、SWCLK、目标板供电 |
| 串口乱码 | 不先改代码 | 电平、TX/RX、共地、波特率、时钟 |
| 模块一工作就复位 | 暂时拔掉模块 | 供电峰值、电源路径、GND |

## 0.5.10 本章要点

- 本书的唯一硬件基线是 STM32F103ZET6；
- 软件背景读者应先把“电压、GND、引脚、供电”当作硬件世界的基础类型系统；
- 接线前先确认电压和共地，出问题先断电；
- ST-Link 负责烧录和调试，USB-TTL 负责观察程序日志；
- 只要能安全上电、识别 ST-Link、理解 TX/RX/GND，第 0 章的工程搭建就有了可靠起点。
- “ZET6 支持某引脚功能”和“我的板子把它接给某个器件”是两类事实；后者必须由原理图、丝印或测量确认。
