# 附录 A · STM32F103ZET6、供电与接线速查

> 本书唯一硬件基线：STM32F103ZET6 开发板。请把本附录当作每次接线前的检查表，而不是只在出错后才翻出来。

## A.1 固定工程参数

| 项目 | 本书配置 |
|---|---|
| 目标芯片 | STM32F103ZET6 |
| 内核 | Cortex-M3 |
| 最大系统时钟 | 72 MHz |
| Flash | 512KB |
| SRAM | 64KB |
| SPL 密度宏 | STM32F10X_HD |
| 启动文件 | startup_stm32f10x_hd.s |
| 链接脚本 | Flash 512KB、SRAM 64KB |
| 调试接口 | SWD + ST-Link |

本书不以 C8T6、Medium Density 或其他容量的链接脚本为主线。遇到其他教程的配置时，先确认它是否适用于 ZET6。

## A.2 本书常用引脚

这些是教材示例常用的默认映射。开发板的板载外设可能使用不同引脚，接线前仍需看丝印、原理图和数据手册。

| 功能 | 常用引脚 | 章节 | 注意事项 |
|---|---|---|---|
| 板载 LED | PC13 | 0、3、7 | 很多板子为低电平点亮 |
| USART1 调试 | PA9 / PA10 | 8 以后 | TX/RX 要与 USB-TTL 交叉 |
| USART2 模块通信 | PA2 / PA3 | 18、21–24 | WiFi/BLE 模块需共地 |
| I2C1 | PB6 / PB7 | 10、25 | SDA/SCL 需要上拉 |
| SPI1 | PA4 / PA5 / PA6 / PA7 | 11、13、25 | 每个从设备独立 CS |
| ADC1 通道 1 | PA1 | 9、25 | 输入电压不能超过模拟参考范围 |
| SWD | SWDIO / SWCLK / GND | 0、28 | 按板卡丝印接 ST-Link |

## A.3 上电和电平规则

- ZET6 的普通 GPIO 以 3.3V 逻辑为基线；不要把未知的 5V 数字输出直接接入 GPIO。
- 两个模块要进行 UART、I2C、SPI、RS485 通信时，先确认是否需要共地。
- 外设模块的“供电电压”与“IO 电平”是两件事；模块可由 5V 供电，不代表 TX/RX 一定是 3.3V。
- 不要同时用 USB、ST-Link 3.3V、外接电源向同一板子随意供电。
- 接线错误、焦味、异常发热时，先断电；不要反复带电试错。

## A.4 常用接线模式

### UART

~~~text
MCU TX  ───→ 模块 RX
MCU RX  ←─── 模块 TX
MCU GND ───── 模块 GND
~~~

### I2C

~~~text
MCU SCL ───── 设备 SCL
MCU SDA ───── 设备 SDA
GND     ───── GND
3.3V    ───── VCC（以模块规格为准）
~~~

SDA/SCL 是开漏总线，通常需要上拉电阻；没有上拉时，设备可能完全没有响应。

### SPI

~~~text
MCU SCK  ───── 所有设备 SCK
MCU MOSI ───── 所有设备 MOSI
MCU MISO ←──── 所有设备 MISO
MCU CS   ───── 每个设备各自一个 CS
GND      ───── GND
~~~

## A.5 接线前最后检查

- [ ] 板子确实是 STM32F103ZET6；
- [ ] 工程使用 HD 启动文件与 512KB Flash 链接脚本；
- [ ] 电源、电平与 GND 已确认；
- [ ] TX/RX 已交叉；
- [ ] I2C 已确认上拉；
- [ ] 外部模块的峰值电流没有超过开发板供电能力；
- [ ] 所有线按引脚标签连接，不按颜色猜测。
