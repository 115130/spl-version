# 附录 C · 调试工具与最小测量方法

> 代码告诉你“程序认为发生了什么”；工具告诉你“电路实际上发生了什么”。

## C.1 工具选择

| 工具 | 最适合解决什么 |
|---|---|
| ST-Link + OpenOCD + GDB | 程序是否跑到某一行、寄存器值、HardFault |
| USB-TTL | 启动日志、AT 命令、错误码 |
| 万用表 | 是否有 3.3V、是否共地、平均电流、通断 |
| 逻辑分析仪 | UART/I2C/SPI/RS485 时序和协议帧 |
| 示波器 | PWM 波形、噪声、短暂电流峰值、模拟信号 |

## C.2 万用表的前三个动作

1. **量电压**：黑表笔接 GND，红表笔接 3.3V 或目标引脚；确认供电是否正常。
2. **量通断**：必须断电后使用；确认杜邦线和面包板孔位是否真的连通。
3. **量电流**：电流表需要串联，不要把表笔像量电压那样直接并到电源两端。

第一次测量前，先在已知的 3.3V 和 GND 上练习，确认档位和表笔位置正确。

## C.3 逻辑分析仪的最低收益

- UART：看 TX/RX 是否有数据、波特率是否正确；
- I2C：看 ACK、地址、SDA 是否被拉低；
- SPI：看 CS 是否切换、时钟极性与相位是否匹配；
- RS485：看收发使能方向是否在正确时刻切换。

逻辑分析仪不能代替万用表：它看得见数字变化，但不擅长告诉你“供电只有 1.8V”。

## C.4 调试的推荐顺序

~~~text
先供电 → 再连线 → 再时钟 → 再串口日志 → 再协议解析 → 最后看业务逻辑
~~~

不要一开始就怀疑复杂代码。硬件零基础时，大部分问题出在供电、GND、TX/RX、引脚复用或时钟。

## C.5 安全边界

- 不接市电，不拆电源适配器；
- 不确定时先断电；
- 电机、舵机、WiFi 模块的峰值电流通常不能靠 MCU 板的 3.3V 小接口硬扛；
- 有锂电、电机或继电器时，先看模块额定参数和隔离方案。
