# ZET6 教材板卡资源约定

> 这是本书的唯一“板卡事实入口”。芯片型号固定为 **STM32F103ZET6**；板载 LED、按键、Flash 和 USB 串口的接线会随开发板批次变化，不能从 MCU 型号反推。

## 1. 不随板卡变化的 MCU 事实

| 项目 | 本书固定值 |
|---|---|
| 芯片 | STM32F103ZET6，LQFP144 |
| SPL 宏 / 启动文件 | `STM32F10X_HD` / `startup_stm32f10x_hd.s` |
| Flash / SRAM | 512KB / 64KB |
| 调试 | SWD：PA13(SWDIO)、PA14(SWCLK) |
| 首选调试 UART | USART1：PA9(TX)、PA10(RX) |
| I2C1 | PB6(SCL)、PB7(SDA) |
| SPI1 | PA5(SCK)、PA6(MISO)、PA7(MOSI)；CS 由板卡决定 |
| DAC | PA4(DAC1_OUT)、PA5(DAC2_OUT) |

## 2. 每次开始实验前必须确认的板卡事实

在原理图、丝印或万用表中记录下面六项。未确认前，正文中的引脚只可当作**外接实验默认值**，不能当作“板载一定如此”。

| 项目 | 你的实际值 | 验证方法 |
|---|---|---|
| LED 引脚与有效电平 | 待填写 | 输出高/低并观察或量引脚 |
| 按键引脚、上拉/下拉与有效电平 | 待填写 | 断电查电路；上电读 IDR |
| USB 串口芯片接哪路 USART | 待填写 | 原理图 / 系统串口枚举 |
| W25Q Flash 的 CS、WP、HOLD | 待填写 | 原理图 / 读 JEDEC ID |
| RS485 的 UART 与 DE/RE | 待填写 | 原理图 / 回环测试 |
| SD 卡槽是否实际接 SDIO | 待填写 | 原理图 / 低速 CMD0 响应 |

## 3. 本书的外接实验默认映射

这些映射不依赖某块板的板载器件；没有冲突时，用杜邦线按表搭建即可。

| 实验 | 默认映射 | 约束 |
|---|---|---|
| 首次 LED | PC13，低有效仅作为常见默认 | 若板载 LED 不在 PC13，修改 `board.h`，不要改每章业务代码 |
| 按键 / EXTI | PA0，外接按键到 GND，使用内部上拉 | 与 TIM2_CH1 输入捕获实验不能同时占用 |
| 电位器 ADC | PA0 / ADC1_IN0 | 只能接 0–VDDA；与按键实验互斥 |
| DS18B20 | 任意普通 GPIO + 外部上拉 | 使用开漏释放；推荐单独选择 PB0 |
| DHT11 | 任意普通 GPIO + 外部上拉 | 是自定义单线时序，不是 Dallas 1-Wire；推荐单独选择 PB1 |
| UART 调试 | PA9/PA10 + 3.3V USB-TTL | TX/RX 交叉，必须共地 |

## 4. 资源冲突规则

1. 同一引脚一次只属于一个实验；章节中出现同名引脚不代表可把两个完整示例直接合并。
2. 同一个定时器一次只配置一种时间基、PWM 或输入捕获角色；组合项目前先填资源表。
3. 只有“已由原理图确认”的板载映射才能写进 `board.h`；其余采用外接实验映射。
4. SPI Flash、SD 卡、RS485、无线模块都必须先独立通过验收，再进入综合项目。

## 5. 最小 `board.h` 约定

所有示例把可变的 LED / 按键定义放在 `board.h`，业务代码只调用 `BoardLed_Write()` 和 `BoardKey_Read()`。换板时只改一处。
