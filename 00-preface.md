# 前言（SPL版）

## 这本书写给谁

你可能是这样的一个人：

- 学过 Java 或 Python，能写像样的程序
- 上过数据结构、操作系统、计算机网络、计算机组成原理这几门课
- C 语言基础过硬——指针、内存、位运算你都不怕
- 但是——**你从来没有碰过嵌入式**
- 你用的是 **Linux 系统**（如 NixOS、Ubuntu、Arch），习惯命令行，不想碰 IDE

每次看到别人用单片机做了个智能小车、温湿度计、WiFi 插座，你觉得很酷。你打开淘宝搜「STM32 开发板」，发现 ¥15 就能买一片，然后你看着那些术语——GPIO、USART、SPI、PWM、NVIC——感觉每个字母都认识，但就是不知道从哪开始。

这本书就是为你写的。**这是 SPL（标准外设库）版本**——用 GCC + Makefile + OpenOCD，从底层寄存器开始，逐步过渡到 HAL 和 RTOS。

## 和别的嵌入式书有什么不同

市面上的嵌入式书籍大致有两种：

**第一种**：大学教材型。第 1 章讲嵌入式定义，第 2 章讲 ARM 架构，第 3 章开始把每个外设的每个寄存器列一遍...读完 500 页，你知道了所有寄存器的名字，但依然不会写一个能联网的温度计。

**第二种**：项目拼盘型。第一章做流水灯，第二章做数码管，第三章做电机...每个项目之间没有递进关系，做完 20 个项目，你依然不知道为什么这么设计，换了块板子就不会了。

**本书的方法是第三种**：

> 用**一个贯穿始终的项目**——从点亮一颗 LED，到最终构建一个能通过 WiFi/蓝牙与云端通信的多协议网关——把所有的知识**串**起来。

你不是在学「第 7 章定时器」，你是在学「怎么让 LED 以精确的频率呼吸」。
你不是在学「第 8 章 UART」，你是在学「怎么让单片机跟 WiFi 模块说话」。
你不是在学「第 21 章 MQTT」，你是在学「怎么让传感器数据出现在手机 App 上」。

**学每一章的那一刻，你都知道「这有什么用」**。

## 为什么另外做一版 SPL

很多嵌入式初学者问我：「HAL 库一上来就给你生成几千行代码，`main.c` 里调了 `MX_GPIO_Init()`，但里面发生了啥完全看不懂。能不能从更底层开始？」

能。这就是 SPL 版存在的意义。

| 你学 HAL 时的心态 | 你学 SPL 时的心态 |
|-------------------|------------------|
| 「CubeMX 帮我配好了，我不需要懂」 | 「我亲手写的初始化代码，每个寄存器值我都知道为什么」 |
| 「`HAL_GPIO_WritePin` 到底做了啥？」 | 「`GPIO_SetBits(GPIOB, GPIO_Pin_5)` 就是往 BSRR 写一位，一步到位」 |
| 「好方便，但感觉浮在表面」 | 「好清晰，我掌控着一切」 |

SPL 给你底气，HAL 给你效率。两样都学了，你才是完整的嵌入式开发者。

## 你需要准备什么

### 硬件

- 1 块 STM32F103ZET6 开发板（本书唯一硬件基线）
- 1 个 ST-Link v2 / DAP-Link 调试烧录器（¥15）
- 1 块面包板 + 若干杜邦线
- 1 块 DX-WF24 WiFi/BLE 模块（或 ESP8266 AT 兼容模块）
- 1 块 HC-05 蓝牙模块
- 传感器：DHT22、BH1750、SSD1306 OLED 屏
- 详见附录 C

### 软件（全部免费，Linux 原生）

- `arm-none-eabi-gcc`（ARM GNU 工具链）
- `openocd`（调试/烧录服务器）
- `make`（构建系统）
- VS Code 或任意编辑器
- 串口调试工具（`minicom` / `screen` / `picocom`）

### 知识储备

- C 语言：指针、结构体、位运算、`volatile`
- 基本的电路常识：电压、电流、电阻、上拉/下拉（不熟没关系，附录 B 有速览）
- 英语：能读数据手册
- Linux 命令行：`cd`、`ls`、`make`、`gcc` 基本使用

## 为什么选 STM32F103

1. **它是最经典的 ARM MCU**。学会 STM32F103，你就能触类旁通所有 Cortex-M 系列（M0/M3/M4/M7）。
2. **资料最多**。无论你遇到什么问题，Google 上一定有答案。
3. **足够简单**。没有多核、没有 MMU、没有 Cache——你能看到和理解 CPU 做的每一件事。
4. **足够强大**。72MHz 主频、ZET6 有 64KB SRAM、512KB Flash——对于学习绰绰有余。
5. **便宜**。¥15 一块，烧了不心疼。

SPL 库虽然官方已停止更新，但它是一套**最透明**的库——每一个函数你都能打开源文件看到寄存器操作。学会 SPL，你再看 HAL 就如同降维打击。

## 本书代码约定

- 开发环境：**Linux + arm-none-eabi-gcc + Makefile + OpenOCD**
- 库：**SPL (STM32F10x_StdPeriph_Lib)**（第 1-13 章），**FreeRTOS + HAL/SPL 混用**（第 14 章起）
- 构建系统：**手写 Makefile**
- 代码注释使用中文

```c
// SPL 代码风格示例
#include "stm32f10x_gpio.h"
#include "stm32f10x_rcc.h"

void LED_Init(void) {
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);

    GPIO_InitTypeDef gpio;
    gpio.GPIO_Pin   = GPIO_Pin_5;
    gpio.GPIO_Speed = GPIO_Speed_50MHz;
    gpio.GPIO_Mode  = GPIO_Mode_Out_PP;
    GPIO_Init(GPIOB, &gpio);
}

void LED_On(void)  { GPIO_ResetBits(GPIOB, GPIO_Pin_5); }
void LED_Off(void) { GPIO_SetBits(GPIOB, GPIO_Pin_5);   }
```
