# 贡献指南

## 硬件基线

本书的正文与示例统一使用 STM32F103ZET6：

- STM32F10X_HD
- startup_stm32f10x_hd.s
- Flash 512KB、SRAM 64KB

请不要在正文示例中混入其他容量、启动文件或引脚假设；若要支持其他板卡，应单独建立迁移说明。

## 提交教材内容时

每个实验章节应包含：

1. 学习目标与前置知识；
2. 接线或硬件前提；
3. 最小可运行步骤；
4. 预期现象；
5. 常见失败与排查；
6. 本章要点与下一章链接。

## UART 级章节质量标准

第 8 章 UART 是本书的详细度参照。新增或重写章节不必机械追求相同行数，但必须形成同样完整的学习闭环：

1. 开头说明本章产出、前置知识和适用的 ZET6/SPL 边界；
2. 把概念与一个真实引脚、寄存器、任务、数据帧或工程文件关联；
3. 给出可检查的实现骨架，并明确哪些部分需按模块/平台手册调整；
4. 说明硬件、电平、内存、并发、协议或安全边界中最容易踩坑的部分；
5. 定义最小验收：读者应观察到什么日志、波形、寄存器或状态；
6. 给出失败现象 → 优先检查项，而不是只写“请检查接线”；
7. 用渐进练习把单元实验连接到下一章或综合项目；
8. 不把未在真实 ZET6/SPL 环境验证的示例称为“已可运行”。

章节长短应由学习任务决定。目标是让读者能独立复现、定位和扩展，不是用重复文字堆页数。

## 提交示例代码时

- 不提交 build 产物或真实密钥；
- 在 README 写明“已编译”“已烧录”或“仅骨架”；
- 说明目标板、工具链版本和接线；
- 优先提供最小可复现工程；
- 任何涉及电机、锂电、继电器或门锁的示例必须标明安全边界。
